# <a name="create-a-request-and-define-search-settings"></a>创建请求并定义搜索设置

**本文**：了解如何在Priva中创建请求以开始执行主题权限请求。 了解数据检索的搜索设置以及如何优化搜索。

在“使用者权限请求管理员”角色组中具有角色的用户，而不是仅限视图 ([查看Priva权](priva-permissions.md)限) 可以在“使用者权限请求”中创建请求。 引导式过程将引导你完成有关查找有关数据主体的个人数据的设置，并开始完成其请求的过程。

## <a name="request-types"></a>请求类型

Priva 主体权利请求支持三种不同类型的请求：

1. **访问**：提供组织在Microsoft 365中保存的数据主体的个人信息摘要。

2. **导** 出：提供包含数据主体个人信息的内容项的摘要和导出文件。 这些项目在查看搜索设置收集的数据时已审阅并标记为 **“包含** ”。

3. **用于跟进的标记列表**：生成在数据评审期间标记的任何文件的摘要，这些文件可能需要在Priva之外执行其他操作。 例如，可能需要根据数据主体的请求帮助删除其个人信息。 可以在[Priva设置](priva-settings.md)中查看包含的标记并为组织设置自定义标记。

## <a name="getting-started-with-your-first-request"></a>第一个请求入门

当你启动使用者权限请求的试用版或订阅时，我们会为使用默认设置的第一个请求提供简单的现用设置。 此设置可帮助你浏览主题权限请求工作流并熟悉其功能。

第一次到达“主题权限请求”页时，顶部会显示一个横幅，上面有 **一个开始** 按钮。 当用户选择此按钮时，将显示一个浮出窗格，其中预填充到名称和电子邮件字段中，并显示所有默认设置。

**使用信息探索请求功能**：根据自己的信息试用主题权限请求可帮助你熟悉和舒适地完成过程的每个阶段。 你将看到默认搜索生成的内容，并可以通过调整搜索设置来练习优化结果。 在 **“收集的数据”** 选项卡上，可以查看右侧预览区域中的项目，并练习修改文本、应用标记、输入笔记和标记要包含或排除的最终报表项 (在 [审阅数据中查找主题权限请求](subject-rights-requests-data-review.md)) 的详细信息。

- 无需使用信息创建第一个请求。 如果已准备好启动对数据主体的请求，请将姓名和电子邮件地址替换为数据主体的信息。

若要接受所有设置并创建请求，请选择 **“创建**”。 该窗格将关闭，你将看到“ **主题权限请求** ”页上列出的新请求。 若要在创建请求之前更改任何默认设置，请选择 **“编辑请求详细信息**”，将你置于 [主题权限请求创建向导](#create-a-request)中。

> [!NOTE]
> 创建的任何请求都将计入试用或付费订阅分配，而不考虑哪个数据主体的信息用于请求。 关闭请求后，将应用标准的 30 天数据保留期。 了解如何更改 [使用者权限请求的保留期](subject-rights-requests-reports.md#retention-periods-for-reports-and-data)。

## <a name="create-a-request"></a>创建请求

按照以下步骤开始执行主题权限请求：

1. 在 [Microsoft Purview 合规性门户](https://compliance.microsoft.com/)中，选择左侧导航栏中的 **Priva 主体权利请求**。

2. 在屏幕右上角，选择 **“创建请求**”。

3. 在 **“数据主题信息** ”页上，输入数据主体的名字和姓氏以及电子邮件地址。 选择 **“添加驻留** ”以选择数据主体的居住国。 选择标识数据主题与组织 (（例如，客户或当前员工) ）相关的选项，然后选择 **“下** 一步”转到下一步。

4. 在 **“位置”** 页上，确定要在何处搜索数据主体的信息。 通过将每个选项旁边的状态切换开关移动到 **On** 位置，选择以下一个或两个位置：

   - **Exchange**：在Exchange邮箱以及单独或组Teams聊天中查找数据。 可以选择搜索组织中的所有Exchange帐户，或选择 **“选择帐户**”，从 **Exchange邮箱** 浮出窗格中选择单个用户。

   - **SharePoint**：在SharePoint站点、OneDrive for Business站点和Teams通道中查找数据。 可以选择搜索组织中的所有SharePoint网站，或选择 **“选择网站**”，从 **SharePoint网站** 浮出窗格中选择单个用户。

> [!TIP]
> 有关识别相应搜索词的帮助，请参阅以下主题：
> - SharePoint站点和 URL：[管理SharePoint管理中心的网站](/sharepoint/manage-sites-in-new-admin-center)提供有关如何对网站进行排序和筛选以及如何搜索SharePoint网站的指导。 使用此功能可在SharePoint **站点** 浮出窗格上的搜索字段中查找要输入的 URL。
> - Teams聊天和频道：[Get-Team](/powershell/module/teams/get-team) 演示如何通过提供特定属性或信息在Microsoft Teams中查找团队。
> - OneDrive站点和 URL：[关于OneDrive URL](/onedrive/list-onedrive-urls#about-onedrive-urls) 提供有关用户OneDrive URL 的正确格式和属性的信息。 使用此方法可帮助你识别搜索中的OneDrive网站。

5. 在“ **定义搜索设置”页上** ，可以通过在各种高级搜索选项中进行选择来更改默认搜索。 在此处，还可以选择在自动返回数据之前先获取估计值。 如果选择这些选项中的任何一个，选择 **“下一步** ”时，你将转到其他屏幕。 在下面 [的“定义搜索设置”中](#defining-search-settings) 获取详细信息。 如果不想更改搜索，请将所有选项留空，然后选择 **“下一步** ”以进入下一阶段。

6. 在 **“选择请求类型** ”页上，选择请求类型： **访问**、 **导** 出或 **标记列表以进行跟进** ([请参阅上面的说明](#request-types)) 。 指示请求是否与数据隐私法规相关。 查看或更改完成截止日期（默认为超过请求创建日期两周）。 然后选择“**下一步**”。

7. 在 **“确认”或编辑此请求页的名称** 上，可以保留或编辑为请求提供的友好名称，并输入可选说明。 然后选择“**下一步**”。

8. 在 **“审阅和完成** ”页上，查看在前面的步骤中输入的内容的摘要。 可以通过在每个部分中选择 **“编辑”链接** 来编辑任何字段。 完成后，选择 **“创建请求**”。

创建请求后，你将看到确认屏幕。 选择 **“完成”** 以返回到主“主题权限请求”屏幕。 新请求将列在请求列表的顶部。 从列表中选择它，开始查看和处理进度阶段。

#### <a name="what-happens-after-your-request-is-created"></a>创建请求后会发生什么情况

我们将根据搜索参数立即开始计算预期查找的数据量的估计值。 在几分钟到一小时内访问请求的详细信息页，查看估算结果以及请求是否已转到下一阶段。 请求将从 **数据估算** 自动进行到 **数据检索** ，但以下情况除外：

 - 如果选择在步骤 5 中首先接收数据 esimtae， **请定义搜索设置**，你将能够查看有关搜索的详细信息，并在检索所有数据之前对搜索进行更改。 如果在 **数据估算** 时暂停，则需要手动推进到 **数据检索** 的下一阶段。
 
 - 如果未选择停止数据估算，但我们预测搜索将生成大量数据，则会在请求顶部看到一条消息栏，其中包含一个链接，用于编辑搜索，然后再继续进行 **数据检索**。

访问 [数据估计和检索](subject-rights-requests-data-retrieval.md)，详细了解这些阶段。

## <a name="defining-search-settings"></a>定义搜索设置

创建主题权限请求时，将根据创建向导的“ **位置** ”页上的选择运行默认搜索。 如果要进行更有针对性的搜索，或者如果要在检索内容项之前选择获取数据估算，则可以在请求创建向导的 **“搜索设置”** 页上进行这些选择。

### <a name="advanced-search-options"></a>高级搜索选项

- **优化搜索**：此选项允许指定额外的属性，以帮助在组织的数据中识别数据主体。 选择此选项后，系统会提示你添加更多搜索参数，下面在 [“优化搜索](#refining-your-search)”中进行了说明。
- **包括由数据主体创作的内容**：此选项将查找由数据主体创作的内容。 示例包括由数据主体创建或上传到SharePoint者的文件。 选择此选项可能会显著增加返回的数据量。
- **包括所有版本的项目**：如果选择SharePoint作为搜索位置，默认搜索查询将仅返回当前版本的SharePoint项。 选中此框将返回当前和所有以前版本的SharePoint项，从而产生大量数据供你查看。

> [!TIP]
> 可以选择上传其他材料，使Priva能够根据确切的数据值标识数据主体。 若要了解详细信息，请参阅 [使用者权限请求的数据匹配](subject-rights-requests-data-match.md)。

### <a name="data-estimate"></a>数据估算

“ **获取估算第一个** ”选项将显示在自动检索数据之前预期要查找的数据量的估计值。 当估算显示在请求的详细信息页上时，可以选择查看搜索结果并预览已发现的项的采样。 如果项目表示预期的结果，则需要选择 **“检索数据** ”，以继续实际检索内容项。 获取有关 [数据估算阶段](subject-rights-requests-data-retrieval.md)的更多详细信息。

## <a name="refining-your-search"></a>优化搜索

如果在 **“搜索设置”** 页上选择 **“优化搜索**”，或者在 **“数据估计**”阶段暂停并选择编辑搜索查询，系统会提示你提供个人属性和条件的详细信息，以进一步定位搜索结果。 可在任一类别或两个类别中进行添加。 完成本部分中的选择后，请求的数据检索将基于搜索设置。

下面所述的详细信息也是在 **数据估算** 阶段暂停并选择编辑搜索查询时需要提供的内容。 

#### <a name="add-personal-attributes"></a>添加个人属性

在文本字段中输入数据主体名称、昵称、电子邮件地址和地址的值。 可以添加其他标识符，例如出生日期或电话号码，文本字段支持多个用分号分隔的条目。 完成后，选择 **“下一步** ”以继续访问 **“条件”** 页。

#### <a name="conditions"></a>条件

选择 **“添加条件** ”按钮可在一系列条件中进行选择，以进一步定位搜索，包括项目名称、发件人和收件人姓名、个人数据类型，以及项目是否在组织外部共享。 文本字段支持用分号分隔的多个条目。 完成后，选择 **“下一步** ”以保存搜索设置并将其进度保存到请求类型设置。
