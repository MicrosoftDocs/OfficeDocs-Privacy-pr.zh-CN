# <a name="privacy-risk-management-policies"></a>隐私风险管理策略

隐私风险管理策略可以帮助你解决对组织很重要的风险方案。 我们的策略模板以培养健全的数据处理实践为中心。  警报可让管理员知道何时检测到策略匹配项，可能需要进一步调查。 电子邮件通知和提示Microsoft Teams帮助用户了解哪些活动会带来隐私风险，使用户能够立即解决问题，并将其指向隐私培训。

若要快速入门，请使用具有默认设置的模板为数据过度表达、数据传输以及数据最小化和方案创建新策略。 还可以自定义模板设置，以创建符合组织需求的策略。

## <a name="policy-template-types"></a>策略模板类型

隐私风险管理有三个策略模板，旨在帮助你解决有关保护个人数据的关键关注领域。 每个模板都有可在快速设置过程中接受的默认设置，或使用引导式进程进行自定义。 创建新策略时，第一个任务是选择下面列出的三个模板之一：

- **数据过度表达**：此策略标识包含个人数据的内容项，而其他人可能无法广泛访问这些数据。 找到匹配项时，可以设置通知，提示内容所有者快速应用保护。

- **数据传输**：此策略可以跨你确定的边界检测个人数据传输，这可能涉及到组织外部的传输，或者跨部门或地理区域进行内部传输。 找到匹配项时，可以设置通知，鼓励发件人撤销对内容的访问权限。

- **数据最小化**：此策略标识包含长期未受影响的个人数据的内容项。 找到匹配项时，可以向内容所有者发送通知，提示他们采取快速操作来保留或删除该项目。

## <a name="quick-setup-using-a-template-with-default-settings"></a>快速设置：将模板与默认设置配合使用

直接从模板创建策略时，将自动为你选择大多数设置，以帮助快速启动和运行。 按照以下步骤使用其中一个模板创建具有默认设置的策略：

1. 在 [Microsoft Purview 合规中心](https://compliance.microsoft.com/)中，在左侧导航中查找Priva 隐私风险管理并选择 **“策略**”。

2. 选择屏幕右上角的 **“创建策略** ”，其中显示一个浮出窗格，其中列出了所有策略创建选项。

3. 找到要创建的策略的类型，并在其卡片中选择 **“创建**”。

4. 浮出窗格包含策略详细信息。 选择 **“视图”设置** 将显示默认设置。 可以从此处编辑设置，这会使你进入下面所述的引导式过程。 若要继续使用默认设置创建策略，只需输入描述性名称，然后选择 **“创建策略**”。

将创建策略，并在“策略 **”页上** 找到它。

策略将在测试模式下开始运行，这意味着不会生成任何警报或通知，并且可以监视其性能。 准备好打开策略后，选择策略并对其进行编辑以启用。

## <a name="custom-setup-guided-process-to-choose-all-settings"></a>自定义设置：用于选择所有设置的引导式进程

自定义策略选项是用于创建策略的引导过程。 首先选择一个模板，然后浏览每个设置以自定义策略。 以下说明提供适用于这三种策略类型的基本设置的详细信息。 如果设置因策略类型而异，我们将链接到特定指令。

按照以下步骤创建策略：

1. 在 [Microsoft Purview 合规性门户](https://compliance.microsoft.com/)中，在左侧导航栏中找到Priva 隐私风险管理。 在下拉菜单中，选择 **“策略**”。

2. 选择“**创建策略**”。

3. 选择 **“自定义** ”选项，使用隐私风险管理中的策略创建向导创建策略。

4. 选择策略类型：**数据过度表达、****数据传输** 或 **数据最小化**。

5. 为策略提供描述性名称，以帮助你在策略列表中标识它。 提供可选说明，然后选择 **“下一步**”。

6. 接下来的步骤允许定义所有策略设置。 有关更多详细信息，可跳转到本文中的说明。 选项包括：
    - [**要监视的数据**](#choose-data-to-monitor)：选择策略将监视的个人数据类型。
    - [**用户和组**](#choose-users-and-groups)：将策略应用到所有用户或所选用户。
    - [**位置**](#choose-locations)：将策略应用于Microsoft 365中的选定区域。
    - [**条件**](#set-conditions)：设置策略的条件。 这些选项因策略类型而异。
    - [**结果**](#define-outcomes-user-email-notifications-and-tips)：定义找到策略匹配项时的结果，例如用户电子邮件通知。
    - [**警报**](#set-alerts)：确定找到策略匹配项时向管理员发出警报的频率。
    - [**模式**](#testing-a-policy)：选择是否先在测试模式下运行策略。
7. 完成所有设置后，请查看你的选择，进行任何所需的编辑，然后选择 **“提交** ”以创建策略。

几秒钟后，你将看到已创建策略的确认。 在确认页上选择 **“完成** ”，该页面将转到“策略 **”页，** 此时会在表顶部看到新策略。

以下各节提供有关每个策略设置的更多详细信息。

## <a name="choose-data-to-monitor"></a>选择要监视的数据

创建或编辑策略时，我们将要求你选择策略应监视的数据类型。 有两个选项：

- **分类组**：敏感信息类型的分组的可搜索列表;例如，基于《澳大利亚健康记录法》的组，或基于美国个人身份信息（如美国护照号码）的组。

- **单个敏感信息类型**：敏感信息类型的可搜索列表;例如，社会保险号码或驾驶执照号码。

如果从现有分类组中进行选择，则也不能选择单个类型或创建自己的组。 为了获得最大的灵活性，请选择单个敏感信息类型。 若要利用最常见的标准，请从分类组中进行选择。 详细了解下面的每种数据类型。

### <a name="classification-groups"></a>分类组

分类组是 [敏感信息类型的](/microsoft-365/compliance/sensitive-information-type-entity-definitions) 分组，用于检测与个人数据或特定法规相关的内容。

在 **“数据”监视** 页上选择此选项时，需要选择 **“+添加分类组** ”，然后从浮出窗格中显示的列表中选择一个或多个组。

### <a name="individual-sensitive-information-types"></a>单个敏感信息类型

通过选择特定 [的敏感信息类型](/microsoft-365/compliance/sensitive-information-type-entity-definitions)（如社会保险号码或驾驶执照信息），可以自定义要查找的自己组或数据组。 可以在隐私风险管理中从敏感信息类型的完整列表中进行选择。 每个信息类型都有其自己的属性。

在 **“数据”监视** 页上选择此选项时，将显示一个选择器， **默认** 值将作为要选择的敏感信息类型的组的名称列出。 保留或编辑此组名称，然后选择 **“添加** ”以从隐私风险管理中的完整列表中选择一个或多个敏感信息类型。 每个信息类型都有其自己的属性和建议的设置，你可以在添加信息类型后选择置信度下拉菜单右侧的信息图标来发现这些设置。 还可以更改每个敏感信息类型的实例计数。 此设置指定要检测策略的每个敏感信息类型的唯一实例数。

如果创建多个组，向导允许你选择组应如何关联 (“和”或“或”关系) 并定义其操作顺序。

## <a name="choose-users-and-groups"></a>选择用户和组

有两个选项用于确定策略将涵盖哪些用户：所有用户和组，或特定用户和组。

- **所有用户和组**：此选项将策略应用于组织中的所有用户和Office 365组。

- **特定用户或组**：此选项允许你选择单个用户、单个Office 365组或两者的组合。
  - **若要选择用户**： **选择“用户** ”，然后在浮出窗格中，通过在搜索框中输入电子邮件地址来搜索用户。 或者从列表中找到用户，然后选中其名称左侧的复选框。 最多可以选择 100 个用户。 完成后，选择 **“添加”。**
  - **若要选择组**： **选择“选择组** ”，然后在浮出窗格中，选中每个组名称左侧的复选框。 最多可以选择 10 个组。 完成后，选择 **“添加**”。

指定用户和组后，选择 **“下一步** ”以转到下一步。

## <a name="choose-locations"></a>选择位置

在此步骤中，你将指定希望策略在Microsoft 365环境中查找个人数据匹配项的位置。 位置选项将取决于策略类型，你可以选择多个策略类型。 下面介绍了每个位置。

- **Exchange**：策略将标识用户Exchange帐户中的匹配项，其中包括电子邮件正文中的内容以及Exchange邮箱发送或接收的附件中的内容。

- **OneDrive**：策略将标识存储在用户OneDrive for Business帐户中的文件中的匹配项。

- **Teams**：策略将在Teams频道和聊天中标识用户邮件中的匹配项。

- **SharePoint**：策略将标识存储在用户SharePoint站点中的文件中的匹配项。 选择此选项时，将在以下选项之间进行选择：
    - **所有SharePoint网站**：此选择将涵盖组织中所有用户的所有网站。

    - **特定SharePoint网站**：此选择要求指定要应用策略的特定网站。 可以直接在 URL 框中输入特定网站的 URL，然后选择 **+** 要将其添加到网站列表的符号。 还可以选择 **“选择网站**”，然后从浮出窗格中，搜索并从有权访问的网站列表中进行选择。 选中将鼠标悬停在要选择的站点上时显示的框。 进行选择后，选择 **“添加**”。  所有所选网站都将列在“ **位置”** 页底部。
    
    > [!TIP]
    > 如果需要帮助确定组织中的SharePoint网站，请访问[SharePoint管理中心中的管理站点](/sharepoint/manage-sites-in-new-admin-center)。

完成指定位置后，选择 **“下一步**”。

## <a name="set-conditions"></a>设置条件

检测策略匹配的条件因策略模板而异。

- **数据过度表达**：请参阅 [数据公开策略自定义设置说明](risk-management-policy-data-overexposure.md#custom-setup-guided-policy-creation-process)中的条件步骤。
- **数据传输**：请参阅 [数据传输策略自定义设置说明](risk-management-policy-data-transfer.md#custom-setup-guided-policy-creation-process)中的条件步骤。
- **数据最小化**：请参阅 [数据最小化策略自定义设置说明](risk-management-policy-data-minimization.md#custom-setup-guided-policy-creation-process)中的条件步骤。

## <a name="define-outcomes-user-email-notifications-and-tips"></a>定义结果：用户电子邮件通知和提示

结果设置在策略创建向导的 **“结果** ”页上处理。 在此页面上，可以选择在用户执行与策略条件匹配的操作时向用户发送电子邮件通知。

数据传输策略有一个附加选项，用于在用户的操作生成策略匹配项时向Teams中的用户显示提示。 这些提示将包括指向你提供的隐私培训的链接，并包括解决潜在风险的机制。

这些通知可以是防止问题升级的有用机会，并建立用户采用安全数据处理做法的技能和信心。

访问 [隐私风险管理中的用户通知](risk-management-notifications.md) ，详细了解如何使用用户通知。

## <a name="set-alerts"></a>设置警报

警报可帮助管理员知道用户事件何时与策略的条件匹配。 设置警报是可选的，你可以控制生成警报的频率、在生成警报之前必须达到的阈值以及警报的严重性。 警报显示在“**策略**”页上的 **“警报**”卡片上。 详细了解 [如何查看、调查和修正警报](risk-management-alerts.md)。

#### <a name="turn-on-alerts"></a>打开警报

首次创建策略时，可以打开警报，或稍后编辑策略以启用警报。 在策略创建向导的 **“警报** ”页上，将 **“创建警报** ”切换切换到 **“打开** ”位置。

#### <a name="alert-frequency-and-thresholds"></a>警报频率和阈值
打开警报后，决定生成警报的频率。

- **每次发生策略匹配时发出警报**：选择此选项可能会产生大量警报。
- **达到特定阈值时发出警报**：将根据检测到的用户事件数和频率设置阈值。

#### <a name="alert-severity-level"></a>警报严重性级别
选择“低”、“中”或“高”严重级别。 建议组织定义每个级别代表你的内容。

## <a name="testing-a-policy"></a>测试策略

创建策略时，默认设置是在测试模式下启动它。 这意味着创建策略后：

- 不会生成任何警报。 但是，检测到匹配项时，你将看到策略详细信息页上的见解，包括检测到的数据类型及其位置。

- 检测到策略匹配项时，不会发送任何用户电子邮件通知。 但是，你将在策略的详细信息页上看到见解，其中显示了哪些用户与策略匹配项相关联。

测试模式允许查找过去 30 天用户活动中的匹配项。 使用这些见解，可以衡量策略的行为，并查看策略打开时可能生成的警报类型。

我们建议测试策略至少五天，以帮助你了解将生成的匹配项的类型和数量。 可以在策略处于测试模式时 [对其进行编辑](#edit-a-policy) ，以便在启用更改之前监视更改对其性能的影响。 例如，你可能会发现策略过于宽泛，其条件需要调整。 或者，你可能会意识到，根据活动，它会检测到警报不会在对你有用的时间范围内生成。

策略的详细信息页指示测试已运行的天数。 你将看到按位置找到多少个匹配项、检测到多少个与策略条件匹配的用户事件，以及策略匹配检测到的个人数据类型。

> [!NOTE]
> 如果 **将测试模式下的运行** 切换到 **“关闭** ”位置，则在创建完策略后 *会打开* 策略。 这意味着，在检测到匹配项时，设置的任何警报或用户通知都将开始生成。

如果对策略的设置感到满意并准备好将其打开，请选择“蓝色 **打开策略** ”按钮。 该策略现在将处于活动状态，并将生成你设置的任何警报和用户通知。

## <a name="turn-on-a-policy"></a>打开策略

创建完策略后，可以设置为立即启用策略。 不建议这样做，因为最好在打开策略之前将策略置于测试模式，以便监视性能和设置 (请参阅“ [测试](#testing-a-policy) 策略”) 。

如果已在测试模式下创建策略，可以按照以下步骤快速启用策略：

1. 在 **“策略”** 页中，找到策略并选择其名称以打开其详细信息页。
2. 在“ **策略”状态** 卡中，选择 **“打开策略**”。

该策略现在将处于活动状态，并将生成你设置的任何警报和用户通知。

## <a name="turn-off-a-policy"></a>关闭策略

可以通过选择“ **关闭** 策略”页面右上角的策略，随时关闭策略。 当策略关闭时，它不会检测匹配项或生成警报或电子邮件通知。 关闭策略不会删除策略。 可以通过选择策略详细信息页面右上角的“ **打开** 策略”来重新打开策略。

## <a name="view-details-and-activity-from-the-policy-details-page"></a>从策略详细信息页查看详细信息和活动

每个策略都有一个详细信息页，显示策略检测到的活动和见解，以帮助你解决风险。

创建策略后，在主“ **策略** ”页的表中选择其名称。 策略详细信息页的“ **概述** ”选项卡将告知策略的状态，提供数据的见解，并突出显示策略匹配项。 可在此处查看有关特定策略匹配项的详细信息，并详细了解后续步骤。 如果策略在测试模式下运行，则会在此页上看到建议的后续步骤，以及打开策略的按钮。

打开策略后，可以继续查看其策略详细信息页，查看有关问题领域、警报严重性和趋势以及采取的纠正措施的持续见解。

### <a name="overview-tab"></a>“概述”选项卡

在策略详细信息页的 **“概述** ”选项卡上，你将找到有关策略检测到的数据类型和位置以及用户活动位置的详细信息。 下面介绍了有关策略详细信息页的见解。 打开策略后，数据可能需要长达 48 小时才能通过。

#### <a name="policy-status"></a>策略状态

策略状态卡将指示策略是否处于以下三种状态之一： **测试**、 **打开** 或 **关闭**。

**测试**：本部分显示策略处于测试模式的天数，这意味着它根据设置的条件查找策略匹配项，但没有生成警报或用户通知。 在开启策略的好时机时，我们将提供建议。 可以通过选择此卡上的 **“打开策略** ”按钮随时打开它。

**打开**：当策略处于打开状态时，状态卡会显示在策略匹配后发生纠正操作时突出显示的指标，这些指标会生成警报和用户通知。

- **用户执行的操作**：此指标显示从发送的通知总数中收到通知电子邮件的提示时，用户执行的修正操作数。 例如，8/10 表示在发送的 10 个用户通知中，用户在 8 个实例中执行了修正操作。

- **用户解析率**：此指标是用户根据生成的通知数执行修正操作的速率。 如果百分比较低，则可能需要 [编辑电子邮件内容](risk-management-notifications.md#preview-and-customize-email-content)，或仔细检查匹配项以确定策略是否检测到预期的活动。

- **执行的管理员操作**：此指标显示当策略生成警报时管理员执行的修正操作数。 详细了解 [如何对警报执行操作](risk-management-alerts.md#manage-alerts)。

- **管理员解析速率**：此指标是管理员根据警报数执行修正操作的速率。

#### <a name="matches-by-location"></a>按位置匹配

“**按位置** 匹配”卡显示策略根据Microsoft 365位置检测到的内容项数。

#### <a name="user-notifications"></a>用户通知

**“用户通知**”卡片显示一个条形图，显示如果启用了这些功能，则策略生成的用户通知电子邮件数。

#### <a name="matches-by-user"></a>按用户匹配

“ **按用户卡匹配** ”列出其操作已触发策略匹配的顶级用户。 你将看到策略为每个用户检测到的事件数，以及从电子邮件通知中执行的修正操作数。 选择 **“查看** 此卡上的所有用户”以查看策略检测到的用户的完整列表。

#### <a name="matches-by-data-type"></a>按数据类型匹配

“ **按数据类型** 匹配”卡片显示策略匹配项检测到的个人数据类型以及每种类型的数量。 饼图有助于直观地演示特定类型的个人数据（例如社会保障号码或信用卡号）是否主要在你试图识别的风险方案中表示。

> [!TIP]
> 全面了解策略匹配项中涉及的位置、数据类型和用户数目时，可以更好地了解帮助组织保护其存储的个人数据所需的培训和数据保护措施类型。

### <a name="matched-items-tab"></a>“匹配项”选项卡

“ **匹配项”** 选项卡显示与策略中设置的条件匹配的所有内容项的列表。 在此视图中，你可以从其行中选择一个项目，以便在项目列表右侧的窗口中对其进行预览。

在预览窗口中，你会发现以下选项卡提供有关每个项的详细信息：
- **源**：显示触发匹配的个人数据。
- **详细信息**：显示组织中的内容所有者 (用户) 项、项目Microsoft 365位置、项目中的个人数据类型数以及特定的个人数据类型。
- **文件活动**：显示应用于该项的任何标签或分类。
- **修正历史记录**：显示有关用户或管理员对该项目执行的修正操作的信息。

## <a name="edit-a-policy"></a>编辑策略

可以随时编辑策略的设置，无论是在测试模式下还是打开。 可以更新大多数策略设置，包括在打开策略后将策略重新置于测试模式。 唯一无法编辑的设置是策略模板和策略名称。

若要编辑策略，请执行以下步骤：

1. 在 [Microsoft Purview 合规性门户](https://compliance.microsoft.com/)中，在左侧导航栏中找到Priva 隐私风险管理。 在下拉菜单中，选择 **“策略**”。

2. 在“策略 **”页上** 选择要从其行中编辑的策略，其中会显示该策略的详细信息页。

3. 在策略详细信息页上，选择页面右上角的 **“编辑** ”命令。 此操作将让你参与隐私风险管理中的策略创建。

4. 继续执行要更改的设置的步骤。 可以编辑除策略模板和策略名称以外的所有设置。 选择 **“下一步** ”以继续执行下一步。

5. 在 **“完成”** 页上，查看设置，然后选择 **“提交** ”以保存所做的更改。

## <a name="delete-a-policy"></a>删除策略

如果需要删除现有的隐私风险管理策略，请在“ **策略** ”页上的列表中找到它，选择操作菜单 (垂直省略号) ，然后选择 **“删除策略** ”操作。 还可以打开策略的详细信息页，并在右上角选择 **“删除** ”。

在最终删除并永久删除策略之前，系统会要求你确认自己的选择。 删除策略不会影响之前由策略评估的任何文件，策略生成的问题和警报仍将列在 **“警报** 和 **问题** ”页面上。
